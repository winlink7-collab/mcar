# אפיון טכני ומוצרי - מערכת ניהול (Admin Panel) לחברת mcar

מסמך זה נועד עבור **צוות הפיתוח המקצועי / מתכנת הבקאנד** של חברת mcar. המטרה היא בניית מערכת ניהול (CMS + CRM) בתצורת פרימיום – שתהיה מצד אחד סופר-קלה לשימוש עבור "הפקיד/הפשוט", ומצד שני גמישה להחריד כדי לאפשר צמיחה אינסופית לעסק. חוויית המשתמש במערכת הניהול חייבת להיות ברמה זהה לאתר עצמו — מודרנית, נקייה, עוצמתית, וללא צורך בידע בקוד.

---

## 1. ארכיטקטורת ממשק הניהול (UI/UX)
ממשק הניהול ייבנה בתצורה מודרנית (למשל עם לוח בקרה שמאלי / Sidebar ניווט, ותצוגת נתונים מרכזית).
- **טכנולוגיית ספריות מומלצת:** תבנית ניהול כמו Tailwind UI / Bootstrap Admin / Vue.js Admin כדי לחסוך זמן פיתוח אך לשדר יוקרה. מומלץ להשתמש ב-DataTables לתצוגת רשימות (מיון, סינון, חיפוש מהיר).
- **עורך תוכן מתקדם (WYSISYG):** הטמעת כלים כמו **TinyMCE** או **QuillJS** בכל אזור טקסט חופשי (עמוד תנאי שימוש, אודות, מאמרים). זה יאפשר למשתמש הפשוט להדגיש טקסט, להוסיף קישורים ותמונות מבלי לגעת ב-HTML.
- **מנהל מדיה (Media Manager):** כל העלאה של תמונה (רכב, אווירה, לוגו) תעבור דרך רכיב חכם שיחתוך אותן, יכווץ לפורמט `WebP` וישמור על ביצועים גבוהים.

---

## 2. מודולים עיקריים (פירוט טבלאות וישויות)

### 2.1. קטלוג רכבים (Inventory Management)
- **רשימת רכבים (Grid View):** צפייה בכל הרכבים עם תמונה קטנה (Thumbnail), מותג, סוג מנוע, זמינות במלאי ומחיר ליסינג נמוך. אפשרות הדלקה/כיבוי (Active / Inactive) של כל רכב.
- **עריכת/הוספת רכב:** המשתמש יוכל להקליד את הפרטים בטופס מסודר:
  - **מידע בסיסי:** יצרן, דגם, רמת גימור.
  - **קטגוריות (Dropdown):** בחירה ממשפחת רכבים (SUV, סדאן, משפחתי).
  - **מפרט טכני:** כוח סוס (HP), צריכת דלק (או טווח חשמלי), תאוצה ב-0 ל-100, מספר מושבים, סוג מנוע.
  - **תמחור:** מחיר ליסינג פרטי, תפעולי, ורכישה מלאה.
  - **תגים מיוחדים:** צ'קבוקס (Checkbox) של `Best Value` (העסקה המשתלמת) או `Verified` (מאומת ומיידי).
  - **תמונה:** שדה העלאת תמונה ראשית עבור הכרטיסייה (במקום קישורי ה-Unsplash הזמניים שיש כלינו כעת).

### 2.2. מחולל הדפים (CMS - Dynamic Editor)
חובה לתת לבעל העסק יכולת ליצור דפים חדשים לחלוטין ולערוך דפים קיימים. יש להקים מנוע `Pages` במאגר הנתונים:
- **דפים סטטיים מרכזיים:** פאנל המציג את הדפים: עמוד הבית, אודות העסק, ליסינג תפעולי, יצירת קשר. מנהל יוכל לשנות כותרות, פסקאות (Hero text) ותמונות אווירה בעמודים אלה.
- **יצירת עמודים חדשים לחלוטין (Pages Feature):** 
  - המשתמש יוכל ללחוץ על "הוסף דף חדש".
  - המערכת תדרוש: URL Slug (למשל: `/about-us` או `/company-car-tax`).
  - כותרת גדולה, ותוכן פתוח לחלוטין באמצעות עורך (Wysiwyg). 
  - הגדרות SEO נפרדות: `Title Tag` ו- `Meta Description` כדי שהעמוד יקודם דרך גוגל בקלות.
- **ניהול שאלות ותשובות (FAQ):** תפריט נפרד בו מקלידים שאלה (Q) ותשובה (A). המערכת תמשוך אותם ישירות לעמוד יצירת קשר/ליסינג עסקי.

### 2.3. מערכת קשרי לקוחות ולידים (Mini CRM)
לא עוד אימיילים רופפים. הפורטל ישמש כמרכזייה:
- **מרכז לידים:** תצוגת טבלה פשוטה של כל מי שפנה דרך "קבל הצעה עכשיו" או "טופס צור קשר".
- **שדות הטבלה:** תאריך ושעה (זיהוי מדויק), שם הלקוח, טלפון, סוג העסקה המבוקשת (פרטי, תפעולי וכו'), ואיזה מזהה רכב הוא בחר בזמן הגשת הטופס המהיר.
- **סטטוס טיפול פשוט:** כפתורי מצב: `חדש`, `בטיפול (ננעל)` ו-`טופל/סירוב`. 
- **Export (ייצוא):** כפתור המאפשר לייצא תמיד את כל הלידים ל-Excel/CSV ולשלוח למנהל מכירות חיצוני במקח הדרוש.
- **התראות מנהל:** ניתן להזין אימייל מועדף ב-"הגדרות מתקדמות", כך שבכל פעם שנוצר ליד במאגר – יישלח אימייל אוטומטי מסודר לסוכן המכירות באמצעות API כגון SendGrid או PHPMailer בסיסי.

### 2.4. הגדרות אתר עממי (Global Settings Site Configurations)
אבל אישיות ומיתוג חייבות להיות נגישות ב-0 ידע טכני:
- **פרטי תקשורת:** מנהל יכול לשנות טלפונים, כתובת כפתור ה-WhatsApp המרכזי של האתר (`050-xxxxxxx`), ואימייל לחזרות רשמיות. כל שינוי מעדכן אוטומטית גם את ההדר וגם את הפוטר בכל האתר.
- **סקריפטים ואנליטיקות:** שדות של "Header scripts" ו "Footer scripts". כך שאם צריך להוסיף 'Google Tag Manager', 'Facebook Pixel' או תוסף קשירת נגישות מיוחד - המנהל יוכל להדביק את הסקריפט לשם ולשמור. שום מפתח לא נדרש עבור משימות שיווק מחלקתיות.
- **SEO גלובלי:** תמונת ברירת המחדל כאשר הקישור נשלח בוואטסאפ (Open Graph Image).

## 3. דרישות טכניות מיוחדות עבור המתכנת (Technical Directives for Developer)

> [!WARNING]
> אזהרת אבטחה (SQLi & XSS):
> - כל תעבורת הנתונים ממערכת הניהול למאגר חייבת להשתמש ב-`PDO Prepared Statements`.
> - השרת צריך לפתוח סשן (Session) באבטחת מחמירה רק בעת כניסה לפאנל הניהול. נדרש חיזוק `CORS` והטמעת תגי CSRF בכל טופס מוגש - למניעת חבלות מבחוץ.

> [!TIP]
> ביצועים ואופטימיזציות (Performance Caching):
> - מכיוון שהאתר קורא למאגר נתונים מסיבי של רכבים (SQL), יש לפתח שכבת "מחבוא" (Cache). כדאי שמערכת ה-PHP תמשוך את המלאי לרמת HTML ואז תשתמש ב-AJAX קליל מבלי להציף את מסד הנתונים בעומסי משתמשים כבדים.
> - תמונות מורמות: הטמע מודול שחותך (Crop/Resize) וממיר תמונות ל-WebP לפני השמירה על הקלאוד (Cloudways) כדי שהמערכת תישאר תמיד קלת משקל ואמינה מהאייפונים החדישים ביותר ועד סמארטפונים ישנים.
